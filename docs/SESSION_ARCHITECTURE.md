# Persistent TUI Sessions — Implementation Spec

## Overview
TUI mode maintains a persistent session across prompts within the same `lx` invocation.
CLI mode (`lx "prompt"`) remains one-shot — no session persistence.

## Session Files

```
~/.local/share/linx/sessions/
├── active/
│   └── a1b2c3d4-e5f6-7890.md    # Running TUI session
├── archive/
│   ├── 2026-03-27_a1b2c3d4.md   # Completed sessions
│   └── 2026-03-27_f9e8d7c6.md
└── crashed/
    └── b2c3d4e5-f6a7-8901.md    # Unclean shutdown (moved from active/)
```

## TUI Lifecycle

```
TUI starts
  └→ Check active/ for orphaned sessions (from crashes)
      ├→ Found orphan?
      │   ├→ Parse session file, show summary:
      │   │     "Previous session found (started 14:30, 5 turns):
      │   │      Last prompt: 'Configure waybar with dark theme'
      │   │      Restore? [y/n]"
      │   ├→ Y: Move from crashed/ back, load messages into Agent, resume
      │   └→ N: Archive orphan to archive/, start fresh
      └→ No orphan: start fresh

  └→ Create session UUID + file in active/
  └→ Create Agent ONCE (load memory, create provider)
      └→ User submits prompt
          ├→ Agent.Chat(prompt) — appends to accumulated messages[]
          ├→ Append turn to session file
          ├→ Memory extraction (async, non-blocking)
          ├→ Reset hasResearched (Option A — per-turn)
          ├→ Check context → warn/compact if needed
          └→ Wait for next prompt
      └→ Ctrl+L (clear session)
          ├→ Archive current session → archive/
          ├→ Final memory extraction
          ├→ Create NEW Agent + NEW session UUID + NEW session file
          └→ Research gate resets (new agent)
      └→ Ctrl+C (clean exit)
          ├→ Archive current session → archive/
          ├→ Final memory extraction
          ├→ Remove from active/
          └→ Exit
```

## Research Gate — Option A (Per-Turn Reset)

Within a single prompt/response cycle, once research is done it carries through.
After the turn completes (final response returned), `hasResearched` resets to `false`.
Next prompt must research again before any destructive ops.

```go
// In Chat(), after returning the final response:
a.hasResearched = false
```

## Context Window Management

### Config
```toml
[profiles.local]
context_window = 32000    # tokens (default 32000)

[behavior]
compact_threshold = 90    # % of context before auto-compact (default 90)
```

### Token Estimation
Rough: `totalChars / 4` (chars-to-tokens ratio, conservative)

### TUI Status Bar
- Show context usage: "🐧 Linx — qwen3.5-122b-a10b [45%]"
- At 75%: yellow warning
- At 90%: red warning + auto-compact

### Compaction Strategy
1. Keep: system prompt + memory + last 3 full turns
2. Summarise: all older turns into one "context summary" message
3. Summary generated by LLM: "Summarise these conversation turns concisely"
4. Replace old turns with summary in messages[]
5. Update session file with compaction marker

## CLI Session Management

### List sessions
```bash
lx --sessions                    # List recent sessions
lx --sessions --last 10          # Show last 10 sessions
```

Output:
```
Recent sessions:

  1. [active]  a1b2c3d4  2026-03-27 14:30  local/qwen3.5  5 turns
     Last: "Configure waybar with dark theme"

  2. [archive] f9e8d7c6  2026-03-27 10:15  cloud/gpt-5.4  12 turns
     Last: "Set up Plymouth boot splash"

  3. [crashed] b2c3d4e5  2026-03-26 22:40  local/qwen3.5  3 turns
     Last: "Install NetworkManager"
```

### Resume a session
```bash
lx --resume a1b2c3d4             # Resume by UUID prefix (partial match OK)
lx --resume 1                    # Resume by list number
```

This launches TUI mode with the session's messages loaded into the Agent.

### Delete sessions
```bash
lx --sessions --clear            # Clear all archived sessions (with confirmation)
```

## Provider Credential Refresh ✅

Implemented in `agent/agent.go` (M29). Both `Chat()` and `ChatTurn()` wrap the
`ChatWithToolsStream` call with `isAuthError()` detection. On auth failure the
agent creates a fresh provider via `providers.NewFromConfig()` and retries once.

```go
resp, err := a.provider.ChatWithToolsStream(...)
if err != nil {
    if isAuthError(err) {
        newProvider, refreshErr := providers.NewFromConfig(&a.cfg.Provider)
        if refreshErr == nil {
            a.provider = newProvider
            resp, err = a.provider.ChatWithToolsStream(...)
        }
    }
    if err != nil { /* emit error, return */ }
}
```

The `isAuthError` helper matches 401, 403, "unauthorized", "forbidden",
"authentication", "invalid api key", and "expired" in the error string.

**Deviation from spec:** The refresh logic lives in `agent/agent.go` rather than
`agent/providers/codex.go`. This makes it provider-agnostic — any provider whose
credentials expire benefits from the retry, not just Codex.

### Deviations from Original Plan

1. **Credential refresh is provider-agnostic** — originally specced for `codex.go`,
   implemented in `agent.go` so all providers benefit.
2. **Compaction uses local summarization** — builds a text summary from old turns
   rather than calling the LLM to summarize, avoiding extra API calls and potential
   auth issues during compaction.
3. **Turn records** — `ChatTurn` returns a `Turn` struct with tool call records,
   which wasn't in the original spec but enables richer session file logging.
